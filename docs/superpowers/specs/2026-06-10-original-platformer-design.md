# 原创手机端平台跳跃游戏设计

## 背景与边界

本项目使用 `Vite + TypeScript + Phaser` 构建一个原创两关横版平台跳跃网页游戏。目标是保留经典平台跳跃的通用玩法体验，并针对手机浏览器优先优化。

明确排除以下内容：

- 不使用任天堂官方角色、名称、贴图、音乐、音效、关卡布局或可识别桥段。
- 不一比一复刻任何官方《超级玛丽》关卡。
- 不克隆或修改 Phaser 引擎源码；Phaser 通过 npm 依赖使用。

## 技术方案

采用 `Vite + TypeScript + phaser npm 包`。

建议目录结构：

```text
super-mario/
  package.json
  index.html
  tsconfig.json
  vite.config.ts
  src/
    main.ts
    scenes/
      BootScene.ts
      PreloadScene.ts
      LevelScene.ts
      HudScene.ts
    game/
      player.ts
      enemies.ts
      collisions.ts
      levelLoader.ts
    levels/
      level1.json
      level2.json
  public/
    assets/
      sprites/
      tiles/
      audio/
      fonts/
```

职责划分：

- `src/main.ts` 启动 Phaser 游戏实例。
- `src/scenes/` 管理启动、资源加载、关卡运行和 HUD。
- `src/game/` 放玩家、敌人、碰撞、关卡加载等可测试逻辑。
- `src/levels/` 使用 JSON 描述两关原创地图。
- `public/assets/` 存放原创资源；第一版可先使用程序绘制或轻量占位资源。

## 产品优先级

第一阶段优先级如下：

1. 手机浏览器横屏可顺畅玩通两关。
2. 打磨跳跃、惯性、碰撞和触屏操作手感。
3. 替换和完善原创手绘童话风角色、场景、动画。
4. 增加更多敌人、道具、机关和计分内容。

## 视觉方向

采用原创手绘童话风：

- 圆润形状、柔和渐变、明亮但不刺眼的色彩。
- 角色、敌人、金币、方块、终点全部原创设计。
- 手机屏幕上保持高对比度和清晰轮廓。
- 第一版允许用程序图形或简单 SVG/PNG 占位，保证玩法闭环优先。

## 关卡设计

游戏包含两关原创关卡，每关目标是手机上 2-5 分钟可通关。

第 1 关为教学关：

- 平地、低平台、少量坑洞。
- 少量收集物。
- 2-3 个慢速巡逻敌人。
- 终点使用原创旗帜、传送门或其他原创标识。

第 2 关为进阶关：

- 更高的平台和连续跳跃。
- 可加入移动平台或弹簧。
- 更多收集物。
- 4-6 个敌人。
- 少量隐藏奖励。

第一版暂不做检查点，避免范围扩大。后续在手感稳定后再加入。

## 核心玩法

- 玩家可左右移动、跳跃、冲刺或执行动作。
- 玩家有生命值或 3 次机会。
- 掉坑或侧面碰撞敌人会受伤或重生。
- 踩到敌人顶部可击败敌人。
- 收集物增加分数。
- HUD 显示分数、生命和当前关卡。
- 到达第 1 关终点后进入第 2 关。
- 第 2 关通关后显示完成页。

## 手机端交互

手机端以横屏为主：

- 竖屏显示旋转设备提示。
- 左下角放左右方向按钮。
- 右下角放跳跃和冲刺/动作按钮。
- 按钮半透明，避免遮挡地面判断。
- HUD 放顶部，避免遮挡玩家和平台。
- Phaser 画布使用固定逻辑分辨率，并按 16:9 自适应缩放。

桌面端作为补充：

- 方向键或 WASD 控制移动。
- 空格键跳跃。
- Shift 或其他按键触发冲刺/动作。

## 测试策略

实现阶段遵循 TDD：先写失败测试，再写最小实现。

优先测试纯逻辑：

- 关卡 JSON 解析。
- 玩家状态变化。
- 分数和生命变化。
- 通关条件。
- 敌人巡逻边界。

Phaser 场景交互使用轻量集成测试覆盖关键状态，不把所有动画细节放进单元测试。

最后需要运行：

- 单元测试。
- 构建命令。
- 浏览器手测手机尺寸、横屏提示和触摸按钮。

## 验收标准

- 手机浏览器横屏可从第 1 关玩到第 2 关并通关。
- 方向、跳跃、收集、敌人伤害、踩敌、掉坑重生、关卡切换可用。
- 不包含任天堂官方素材、名称、关卡布局或音乐。
- 测试命令和构建命令通过。
- TypeScript 使用明确类型，不使用 `any`、`@ts-ignore` 或 `eslint-disable`。

## 实现顺序

1. 初始化 Vite + TypeScript + Phaser 项目。
2. 编写关卡数据和核心逻辑测试。
3. 实现关卡加载、玩家控制、碰撞与 HUD。
4. 实现手机触屏控制和横屏适配。
5. 补齐两关原创关卡数据和通关流程。
6. 运行构建验证和浏览器手测。
